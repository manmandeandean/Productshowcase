1. Real-time Search Implementation

The search functionality filters products in real-time as users type, with debouncing to prevent excessive API calls.

// Debounced search to prevent excessive filtering
                                        document.getElementById('searchInput').addEventListener('input', debounce(filterProducts, 300));
    
                                        function debounce(func, wait) {
                                            let timeout;
                                            return function(...args) {
                                                clearTimeout(timeout);
                                                timeout = setTimeout(() => func.apply(this, args), wait);
                                            };
                                        }

How it works: The debounce function delays the execution of the filter function by 300ms, ensuring it only runs after the user stops typing.

2. Dynamic Category Filtering

Category buttons allow users to filter products by type, with visual feedback for the active filter.

// Category filter event listeners

3. Async Product Loading

This function simulates loading product data from a server using async/await and displays a loading message during the delay.

async function loadProducts() {
                                        showLoadingState();
                                        try {
                                            await new Promise(resolve => setTimeout(resolve, 1000));
                                            if (simulateErrorMode) throw new Error("Simulated error");
                                            allProducts = [...sampleProducts];
                                            filterProducts();
                                        } catch (err) {
                                            showErrorState(err.message);
                                        } finally {
                                            simulateErrorMode = false;
                                        }
                                    }

How it works: This function is called when the page loads or when the user clicks the “Reload” button. It uses async/await with a setTimeout wrapped in a Promise to simulate a delay, like waiting for data from a server.

If the simulateErrorMode flag is set to true, it throws a fake error, which is caught and shown to the user. If no error occurs, the sample product data is copied into the main product list and displayed.

4. Product Filtering Logic

This function filters the product list based on both the selected category and the user's search input.

function filterProducts() {
                                        const searchTerm = document.getElementById('searchInput').value.toLowerCase();
                                        let filtered = allProducts.filter(p => 
                                            (currentCategory === 'all' || p.category === currentCategory) &&
                                            (p.name.toLowerCase().includes(searchTerm) || p.description.toLowerCase().includes(searchTerm))
                                        );
                                        if (filtered.length === 0) {
                                            showEmptyState();
                                        } else {
                                            displayProducts(filtered);
                                        }
                                    }

How it works: This function runs every time a user types in the search bar or selects a filter button. It checks two conditions for each product:

  • The product matches the selected category, or the category is “All”
  • The product name or description includes the typed search term

If no products match these criteria, an empty state message is shown. If matches are found, the filtered products are passed to displayProducts() to update the page.

5. Dynamic Product Display

This function dynamically creates and displays product cards on the webpage based on the filtered data.

                                    function displayProducts(products) {
                                        const container = document.getElementById('productsContainer');
                                        hideAllStates();
                                        container.innerHTML = '';
                                        products.forEach(product => {
                                            const card = document.createElement('div');
                                            card.className = 'col-md-6 col-lg-4';
                                            card.innerHTML = `
                                                <div class="product-card">
                                                    <div class="product-image">
                                                        <i class="fas fa-${product.icon}"></i>
                                                    </div>
                                                    <div class="product-info">
                                                        <div class="product-category">${product.category}</div>
                                                        <h5>${product.name}</h5>
                                                        <p>${product.description}</p>
                                                        <strong>$${product.price}</strong>
                                                    </div>
                                                </div>
                                            `;
                                            container.appendChild(card);
                                        });
                                    }

How it works: When this function is called, it first hides any loading, error, or empty state messages using hideAllStates(). Then, it clears the current content of the product container. For each product in the provided list, it creates a new card using document.createElement and fills it with product details like name, description, category, price, and icon. These cards are then added to the page, allowing the user to see only the relevant products based on the filter or search results.

6. State Management System

The application manages multiple UI states: loading, error, empty, and content display.

Loading State

Displays a spinner while data is being fetched, providing user feedback during wait times.

function showLoadingState() {
                                                hideAllStates();
                                                document.getElementById('loadingState').style.display = 'block';
                                            }
Error State

Shows error messages when data loading fails, with retry functionality.

function showErrorState(message) {
                                            hideAllStates();
                                            document.getElementById('errorMessage').textContent = message;
                                            document.getElementById('errorState').style.display = 'block';
                                            }

Displayed when no products match the current search/filter criteria.

7. Best Practices Implemented

  • Separation of Concerns: HTML structure, CSS styling, and JavaScript functionality are clearly separated
  • Event Delegation: Efficient event handling for dynamic content
  • Error Handling: Graceful degradation when operations fail
  • Performance Optimization: Debouncing prevents excessive function calls
  • User Experience: Clear visual feedback for all user actions
